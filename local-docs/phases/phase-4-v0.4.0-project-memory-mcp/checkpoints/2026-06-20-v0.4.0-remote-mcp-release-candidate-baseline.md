## Security Review Summary

- Remote trust decisions remain local-config-only through `.soba/mcp.json` `trustMode`; MCP annotations and metadata do not lower trust.
- Remote tool names are normalized before exposure as OpenAI-compatible function names.
- Remote outputs use the same normalization/truncation path as stdio MCP outputs.
- Static env auth and configured headers are redacted from diagnostics and session output.
- OAuth token diagnostics redact access and refresh token values.
- `MCP-Session-Id` is transport-controlled and rejected from user config headers.

## Known Limitations

- Deprecated HTTP+SSE is not supported as a first-class transport.
- Marketplace discovery, signed MCP servers and SOBA-as-MCP-server export remain out of scope.
- Remote examples that require hosted providers or external packages are templates and disabled by default.
- OAuth discovery, login, token storage, refresh and auth-required UX are verified; direct automatic OAuth bearer injection into every Streamable HTTP request remains a hardening follow-up.
- `.soba/mcp.json` is loaded at session start; hot reload is not supported in v0.4.0.
