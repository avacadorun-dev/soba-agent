## Decisions

- Keep Project Memory injection bounded at 2,000 estimated tokens per turn in `AgentLoop`.
- Use the current user prompt as the relevance query for Project Memory capsules.
- Register memory tools in the main CLI ToolRegistry so the model can explicitly inspect/update memory in addition to passive prompt injection.
- Keep release-verified MCP examples local/deterministic; external package templates stay documented but not release-gated.

## Risks / follow-ups

- Project Memory initialization now happens during CLI startup. If future filesystem-permission errors appear, add a controlled warning/degraded mode rather than silently ignoring memory.
- Memory injection budget is fixed for v0.4.0. Future work should make it configurable once real project traces show pressure patterns.
