## UC-2.5.1: ProviderRegistry — переключение провайдера

**Актор:** Пользователь SOBA
**Предусловие:** Настроен хотя бы один API ключ

**Основной сценарий (/model set):**
1. Пользователь вводит `/model set deepseek/deepseek-chat`
2. Система проверяет наличие API ключа DEEPSEEK_API_KEY
3. Система переключает клиент на DeepSeek
4. Отправляется тестовый запрос (ping)
5. Появляется уведомление "Switched to DeepSeek / deepseek-chat"
6. Конфигурация сохраняется

**Альтернативный сценарий (нет ключа):**
1. Пользователь вводит `/model set kimi/kimi-k2-0711-preview`
2. Система не находит MOONSHOT_API_KEY
3. Система предлагает ввести ключ
4. Пользователь вводит ключ
5. Провайдер активируется

**Исключение (неверная модель):**
1. Пользователь вводит `/model set deepseek/nonexistent-model`
2. Система показывает "Model not found"
3. Текущая модель не меняется

**Критерии приёмки:**
- 4 предустановленных провайдера (deepseek, kimi, alibaba, openrouter) — список B1d
- Проверка API ключа перед переключением
- Тестовый запрос после переключения
- Сохранение конфига
- Уведомление об успехе/ошибке
- `contextWindow` / `maxOutputTokens` берутся из активной модели (B1e), не из плоского конфига
- `selectedModels` в реестре не персистится (B1e) — каталог приходит из runtime discovery

## UC-2.5.2: Визуальные уведомления (Notifications)

**Актор:** Пользователь SOBA
**Предусловие:** TUI запущен, агент активен

**Основной сценарий:**
1. Система инициирует событие (компакция, смена модели, ошибка, активация скила)
2. В правом нижнем углу TUI появляется уведомление
3. Уведомление содержит иконку, заголовок и краткое сообщение
4. Success/info исчезают через 5 секунд
5. Warning/error исчезают через 10 секунд
6. Максимум 3 видимых уведомления одновременно

**Альтернативный сценарий (Escape):**
1. Пользователь нажимает Escape при активном уведомлении
2. Уведомление немедленно исчезает

**Исключение (overflow):**
1. Появляется 4-е уведомление
2. Самое старое скрывается (остаётся в истории)

**Критерии приёмки:**
- Уведомления рендерятся в < 5ms
- Не блокируют интерактивность TUI
- Цвет/иконка по типу (✓/⚠/✗/ℹ)
- Escape закрывает уведомление
- История через `/notifications`

## UC-2.5.3: Понятный диалог доверия (TrustDialog)

**Актор:** Пользователь SOBA
**Предусловие:** Режим permissions = "ask", dangerous-операция

**Основной сценарий:**
1. Агент пытается выполнить опасную операцию
2. TUI показывает модальное окно с предупреждением
3. Доступны кнопки: Allow Once, Allow Session, Allow Repo, Deny (фокус по умолчанию)
4. Пользователь Tab для навигации, Enter для выбора
5. Диалог закрывается, операция выполняется/блокируется

**Альтернативный сценарий (клавиши):**
1. Пользователь вводит y/s/r/n как раньше
2. Диалог распознаёт ввод и закрывается

**Альтернативный сценарий (Escape):**
1. Пользователь нажимает Escape
2. Диалог закрывается с результатом "deny"

**Критерии приёмки:**
- Модальный overlay с затемнением
- Tab-навигация работает
- Escape = Deny
- y/s/r/n обратная совместимость
- Время рендера < 5ms

## UC-2.5.4: Меню выбора модели (ModelSelector)

**Актор:** Пользователь SOBA
**Предусловие:** ProviderRegistry инициализирован

**Основной сценарий (/model list):**
1. Пользователь вводит `/model list`
2. TUI показывает список провайдеров с моделями
3. Текущая модель отмечена (●)
4. Пользователь выбирает модель Enter

**Альтернативный сценарий (поиск):**
1. Пользователь начинает печатать в поле поиска
2. Список фильтруется по имени модели/провайдера
3. Пользователь выбирает модель

**Альтернативный сценарий (discovery errors):**
1. Первый запуск, discovery ещё не выполнялся
2. Selector показывает synthetic-запись из `defaultModel` (если задан)
3. Пользователь может выбрать synthetic-модель и начать работу — discovery запустится позже
4. При следующем открытии selector'а discovery наполнит список реальными моделями

**Исключение (discovery упал):**
1. `/v1/models` вернул таймаут, 401, 403 или DNS-ошибку
2. Selector показывает только synthetic-записи из `defaultModel`
3. Статус-бар отображает предупреждение ⚠
4. Пользователь может переоткрыть selector для повторной попытки discovery

**Исключение (нет /v1/models endpoint):**
1. Провайдер не поддерживает `/v1/models`
2. Selector показывает synthetic-запись из `defaultModel`
3. Другие модели недоступны — только custom `models[]` при добавлении

**Критерии приёмки:**
- Группировка по провайдерам
- Размер контекстного окна виден
- Поиск фильтрует результаты
- Enter выбирает, Escape закрывает
- Synthetic-записи при отсутствии discovery
- Retry discovery без перезапуска TUI
- Ошибки discovery не блокируют UI

## UC-2.5.5: Улучшенный сайдбар (Enhanced Sidebar)

**Актор:** Пользователь SOBA
**Предусловие:** TUI запущен

**Основной сценарий:**
1. Пользователь видит улучшенный сайдбар
2. **Context** — progress bar занятости контекста
3. **Session** — токены, turns, checkpoint
4. **Active Skills** — список активных скилов
5. **Model** — текущая модель с кнопкой Switch
6. **Permissions** — текущий режим с кнопкой Change

**Альтернативный сценарий (сворачивание):**
1. Пользователь нажимает на заголовок секции
2. Секция сворачивается/разворачивается

**Критерии приёмки:**
- Context bar: зелёный <60%, жёлтый 60-85%, красный >85%
- Active Skills отображают реальное состояние
- Model кликабельна → ModelSelector
- Секции сворачиваются/разворачиваются

## UC-2.5.6: Collapsible Tool Results

**Актор:** Пользователь SOBA
**Предусловие:** Есть история сообщений с результатами тулов

**Основной сценарий:**
1. Результат read — показывает path, размер, строки. Контент свёрнут.
2. Результат edit — path, кол-во изменений, diff-подсветка.
3. Результат bash — команда, exit code, output. Если exit != 0 — развёрнут.
4. Пользователь нажимает Enter на заголовке — toggle сворачивания.

**Критерии приёмки:**
- read: path, размер, строки свёрнуты
- edit: diff-подсветка
- bash с exit != 0 развёрнут
- Enter сворачивает/разворачивает

## UC-2.5.7: Turn Visual Separator

**Актор:** Пользователь SOBA
**Предусловие:** Минимум 2 turns в истории

**Основной сценарий:**
1. Между turns отображается разделитель
2. "───── Turn N ─────" с номером
3. Цвет чередуется между turns
4. Возможность свернуть turn

**Критерии приёмки:**
- Разделитель виден между всеми turns
- Номер turn корректен
- Цвета чередуются
- Сворачивание работает

## UC-2.5.8: Search Overlay

**Актор:** Пользователь SOBA
**Предусловие:** Есть история сообщений

**Основной сценарий:**
1. Пользователь нажимает Ctrl+F (или `/search`)
2. Появляется overlay поиска
3. Пользователь вводит запрос
4. Результаты с контекстом
5. ↑/↓ для навигации, Enter — прыжок, Escape — закрытие

**Критерии приёмки:**
- Поиск по всем сообщениям
- Результаты < 100ms для 1000 сообщений
- Навигация ↑/↓ работает
- Enter прыгает к сообщению
- Escape закрывает
