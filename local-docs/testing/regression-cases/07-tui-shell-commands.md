# Регресс-кейсы: TUI — shell-команды (! и !!)

## Цель
Проверить выполнение shell-команд из TUI — `!cmd` (с выводом) и `!!cmd` (без вывода).

## Окружение
- TUI запущен (`.soba -i`)
- Права на выполнение команд в текущей директории

---

## Кейс 01: `!echo` — простая команда с выводом

**Шаги:**
1. Ввести `!echo hello world`
2. Enter

**Ожидаемый результат:** Команда выполнена, вывод `hello world` показан в transcript.

**Критерий PASS:** В transcript появился блок "hello world".

---

## Кейс 02: `!!echo` — команда без вывода

**Шаги:**
1. Ввести `!!echo secret`
2. Enter

**Ожидаемый результат:** Команда выполнена, но вывод НЕ показан в transcript.

**Критерий PASS:** Нет блока с "secret" (хотя команда выполнена — проверить side effect).

---

## Кейс 03: `!pwd` — получение текущей директории

**Шаги:**
1. Ввести `!pwd`

**Ожидаемый результат:** Вывод абсолютного пути текущей директории.

**Критерий PASS:** Путь корректен.

## Кейс 04: `!ls` — листинг директории

**Шаги:**

**Ожидаемый результат:** Список файлов в текущей директории.

**Критерий PASS:** Список не пуст, содержит package.json.

1. Ввести `!git status --short`

**Ожидаемый результат:** Вывод статуса git (изменённые файлы или "nothing to commit").

**Критерий PASS:** Статус показан.

**Шаги:**

## Кейс 13: `!!chmod +x test.sh` — dangerous команда (без вывода)

**Шаги:**
1. Создать `test.sh`
2. `!!chmod +x test.sh`

**Ожидаемый результат:** Команда выполнена, вывод скрыт.

**Критерий PASS:** `test.sh` стал исполняемым.

---

## Кейс 14: `!rm -rf /tmp/test-dir` — dangerous команда (с выводом)

**Шаги:**
1. `mkdir -p /tmp/test-dir`
2. Ввести `!rm -rf /tmp/test-dir`

**Ожидаемый результат:** Trust Manager проверяет: запрос подтверждения.

**Критерий PASS:** Запрос подтверждения (y/n/s/r).

---

## Кейс 15: `!!rm -rf /tmp/test-dir` — dangerous, без вывода

**Шаги:**
1. `mkdir -p /tmp/test-dir`
2. Ввести `!!rm -rf /tmp/test-dir`

**Ожидаемый результат:** Trust Manager проверяет (даже для !!).

**Критерий PASS:** Запрос подтверждения.

---

## Кейс 16: `!sudo ls` — суперпользователь

**Шаги:**
1. Ввести `!sudo ls /root`

**Ожидаемый результат:** Запрос пароля sudo (через терминал) или Trust Manager проверяет.

**Критерий PASS:** Интерактивный запрос sudo (или Trust Manager).

---

## Кейс 17: `!$(echo rm -rf /)` — инъекция

**Шаги:**
1. Ввести `!$(echo echo) test`

**Ожидаемый результат:** Выполняется безопасно (echo test).

**Критерий PASS:** Команда не опасна, вывод "test".

---

## Кейс 18: `!` во время ответа агента

**Шаги:**
1. Отправить агенту "Напиши большой код"

**Ожидаемый результат:** Shell-команда выполняется сразу, не попадая в очередь агента.

**Критерий PASS:** `pwd` выполнен до завершения ответа агента.

---

**Ожидаемый результат:** Shell-команда `exit` выполняется (выходит из TUI? или просто показывает "exit"?).

**Критерий PASS:** Команда не закрывает TUI (проверить поведение).

**Критерий PASS:** Git log показывает коммит.
