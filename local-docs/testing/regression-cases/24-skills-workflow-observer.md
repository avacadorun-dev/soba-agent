# Регресс-кейсы: Skills — workflow observer

## Цель
Проверить, что WorkflowObserver запущен и собирает данные, но не предлагает автоматическое создание skill (v0.3 limitation).

## Окружение
- `.soba -i`

---

## Кейс 01: Observer запущен как core-модуль

**Шаги:**
1. Запустить SOBA с `--debug`
2. Выполнить 2-3 turn'а
3. Проверить debug-логи

**Ожидаемый результат:** В логах присутствуют записи от WorkflowObserver (или компонента наблюдения).

**Критерий PASS:** Observer активен.

---

## Кейс 02: Observer собирает данные о completion gate

**Шаги:**
1. Выполнить несколько turn'ов (некоторые с tool calls, некоторые без)
2. Проверить debug-логи

**Ожидаемый результат:** Observer анализирует completion gate — сколько итераций, какие tool calls, stalled ли.

**Критерий PASS:** Данные собираются.

---

## Кейс 03: Observer не предлагает создать skill

**Шаги:**
1. Выполнить повторяющуюся задачу (например, 3 раза подряд "прочитай файл")
2. Наблюдать

**Ожидаемый результат:** WorkflowObserver НЕ выводит предложение "Создать skill для этой задачи?".

**Критерий PASS:** Нет предложений.

---

## Кейс 04: Observer не мешает работе TUI

**Шаги:**
1. Выполнить 10 turn'ов
2. Наблюдать

**Ожидаемый результат:** Observer работает в фоне, не замедляет TUI, не выводит сообщения.

**Критерий PASS:** TUI отзывчив.

---

## Кейс 05: Observer не сохраняет данные (v0.3 — без персистентности)

**Шаги:**
1. Выполнить turn'ы
2. Перезапустить SOBA
3. Выполнить ещё turn'ы

**Ожидаемый результат:** Observer не имеет cross-session памяти (данные не переносятся).

**Критерий PASS:** Нет ошибок.
