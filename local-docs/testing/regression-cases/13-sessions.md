# Регресс-кейсы: Сессии (session, format, resume)

## Цель
Проверить создание, сохранение, восстановление сессий в JSONL-формате.

## Окружение
- `.soba` собран
- `~/.soba/sessions/` существует

---

## Кейс 01: One-shot создаёт сессию

**Шаги:**
1. `.soba "Привет"`
2. Проверить `~/.soba/sessions/`

**Ожидаемый результат:** Создана новая директория с JSONL-файлом.

**Критерий PASS:** Директория существует, внутри `.jsonl` файл.

---

## Кейс 02: JSONL — валидный JSON в каждой строке

**Шаги:**
1. Выполнить one-shot запрос
2. `cat ~/.soba/sessions/*/*.jsonl | while read line; do echo "$line" | python3 -m json.tool > /dev/null || echo "INVALID"; done`

**Ожидаемый результат:** Нет "INVALID".

**Критерий PASS:** Каждая строка — валидный JSON.

---

## Кейс 03: JSONL содержит все типы

**Шаги:**
1. `.soba "Создай файл session-test.txt"`
2. Прочитать JSONL

**Ожидаемый результат:** JSONL содержит:
- `type: "message"` (user + assistant)
- `type: "function_call"` (write)
- `type: "function_call_output"` (результат write)

**Критерий PASS:** Все 4+ item'а присутствуют.

---

## Кейс 04: `-c` (continue) — продолжение сессии

**Шаги:**
1. `.soba "Моё любимое число 42"`
2. `.soba -c "Какое моё любимое число?"`

**Ожидаемый результат:** Агент отвечает "42".

**Критерий PASS:** Ответ ссылается на предыдущий контекст.

---

## Кейс 05: `-r` (resume) — выбор сессии

**Шаги:**
1. Создать 2-3 сессии разными запросами
2. `.soba -r`

**Ожидаемый результат:** Список сессий для выбора (с ID и/или первыми символами).

**Критерий PASS:** Интерактивный выбор.

---

## Кейс 06: `-s <session-id>` — сессия по ID

**Шаги:**
1. `.soba "Привет"`
2. Взять ID из JSONL (первый item имеет `id`)
3. `.soba -s <id> "Продолжи"`

**Ожидаемый результат:** Контекст восстановлен.

**Критерий PASS:** Ответ учитывает историю.

---

## Кейс 07: `-s <prefix>` — по уникальному префиксу

**Шаги:**

## Кейс 08: `-s` с неоднозначным префиксом

## Кейс 11: `/session` — статистика

**Шаги:**
1. `.soba -i`
2. Отправить 2 сообщения
3. `/session`

**Ожидаемый результат:**
- Format: jsonl
- Active branch: <id>
- Effective tokens: N
- Historical tokens: M (M >= N)
- Hard limit: X
- Context window: Y
- Capsules: Z

**Критерий PASS:** Все поля присутствуют.

## Кейс 14: Сессия с tool call'ами в JSONL

**Шаги:**
1. `.soba "Создай файл session-jsonl.txt с текстом 'test'"`

**Ожидаемый результат:** JSONL содержит:
- user message: `"Создай файл..."`
- function_call: `write` с аргументами
- function_call_output: результат
- assistant message: ответ

**Критерий PASS:** Все 4+ item'а.

---

## Кейс 15: Загрузка большой сессии (1000+ items)

**Шаги:**
1. Создать или иметь сессию с 1000+ строками

**Критерий PASS:** Загрузка не превышает 5 секунд.
