# Регресс-кейсы: Completion Gate

## Цель
Проверить механизм gate — детекцию полного/неполного ответа, stop reason, управление циклом.

## Окружение
- `.soba -i` с `--debug`

---

## Кейс 01: Gate завершает цикл при полном ответе

**Шаги:**
1. `.soba "Ответь 'ok'"`

**Ожидаемый результат:** Агент отвечает "ok" и завершает цикл.

**Критерий PASS:** Только 1 итерация.

---

## Кейс 02: Gate продолжает при tool call

**Шаги:**
1. `.soba "Создай файл gate-test.txt"`

**Ожидаемый результат:** Assistant message → tool call (write) → tool call output → assistant message.

**Критерий PASS:** Gate не завершил после tool call.

---

## Кейс 03: Gate детектирует stop_reason: "max_tokens"

**Шаги:**
1. `.soba --max-tokens 10 "Напиши большое сочинение"`

**Ожидаемый результат:** Ответ обрезан, gate видит stop_reason = max_tokens.

**Критерий PASS:** Ответ обрезан.

---

## Кейс 04: Gate детектирует incomplete (content filter)

**Шаги:**
1. Если модель может вернуть content_filter stop reason — gate продолжает.

**Критерий PASS:** (Требует провайдера с content filter.)

---

## Кейс 05: Gate не бесконечен при ошибке модели

**Шаги:**
1. `.soba --model nonexistent "Тест"`

**Ожидаемый результат:** Gate завершает с ошибкой, не зацикливается.

**Критерий PASS:** Не бесконечный цикл.

---

## Кейс 06: Gate обрабатывает пустой response

**Шаги:**
1. Мок: модель возвращает пустой ответ

**Ожидаемый результат:** Gate детектирует пустой ответ и завершает (или retry).

**Критерий PASS:** Не зависает.

---

## Кейс 07: Gate передаёт context capsules в следующий turn

**Шаги:**
1. После компакции gate включает capsule в следующий system prompt.

**Ожидаемый результат:** Capsule в промпте.

**Критерий PASS:** Контекст сохранён.

---

## Кейс 08: Gate и background compaction не конфликтуют

**Шаги:**
1. Background compaction запускается после turn
2. Gate обрабатывает следующий turn

**Ожидаемый результат:** Нет race condition.

**Критерий PASS:** Работает.
