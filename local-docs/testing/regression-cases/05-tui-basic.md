# Регресс-кейсы: TUI — базовая функциональность

## Цель
Проверить запуск интерактивного режима, отправку сообщений, получение ответов, базовые команды.

## Окружение
- Терминал с поддержкой 256 цветов и Unicode
- Размер окна не менее 80×24

---

## Кейс 01: Запуск TUI

**Шаги:**
1. `.soba -i`

**Ожидаемый результат:**
- TUI запущен
- Строка ввода активна (виден prompt `>`)
- Status bar внизу (модель, язык, тема)
- Transcript область пуста

**Критерий PASS:** Интерфейс рендерится, нет ошибок.

---

## Кейс 02: Отправка сообщения

**Шаги:**
1. `.soba -i`
2. Ввести "Привет" и нажать Enter

**Ожидаемый результат:**
- Сообщение появляется в transcript
- Индикатор загрузки (spinner или "Thinking...")
- Ответ от агента появляется в transcript
- Строка ввода снова активна

**Критерий PASS:** Ответ получен, transcript обновлён.

---

## Кейс 03: Отправка нескольких сообщений последовательно

**Шаги:**
1. `.soba -i`
2. Отправить "Расскажи о себе"
3. Дождаться ответа
4. Отправить "Что ты умеешь?"
5. Дождаться ответа

**Ожидаемый результат:** Оба сообщения и ответы в transcript. Второй ответ учитывает контекст первого.

**Критерий PASS:** Transcript показывает 4 блока (2 user + 2 assistant).

---

## Кейс 04: Ctrl+C во время ответа

**Шаги:**
1. `.soba -i`
2. Отправить "Напиши очень длинный ответ"
3. Во время генерации нажать Ctrl+C

**Ожидаемый результат:**
- Генерация остановлена
- Transcript показывает частичный ответ
- Строка ввода активна
- TUI не закрыт

**Критерий PASS:** TUI остаётся активным после прерывания.

---

## Кейс 05: Новое сообщение после Ctrl+C

**Шаги:**
1. Выполнить Кейс 04
2. Отправить "Продолжай"

**Ожидаемый результат:** Новый ответ от агента (с учётом предыдущего частичного контекста).

**Критерий PASS:** Ответ получен.

---

## Кейс 06: /clear — очистка transcript

**Шаги:**
1. Отправить 2-3 сообщения
2. Ввести `/clear`

**Ожидаемый результат:** Transcript очищен, но сессия не удалена (история сохранена).

**Критерий PASS:** Экран чистый, `/budget` показывает токены.

---

## Кейс 07: /exit — выход из TUI

**Шаги:**
1. `.soba -i`
2. Ввести `/exit`

**Ожидаемый результат:** TUI закрыт, процесс завершён, exit code 0.

**Критерий PASS:** Возврат в shell, exit code 0.

---

## Кейс 08: /help — справка

**Шаги:**
1. `.soba -i`
2. Ввести `/help`

**Ожидаемый результат:** Список всех slash-команд с кратким описанием.

**Критерий PASS:** Содержит: `/exit`, `/clear`, `/help`, `/lang`, `/theme`, `/compact`, `/capsule`, `/rewind`, `/session`, `/budget`, `/auto-compact`, `/config`, `/permissions`, `/queue`, `/skill`, `/project-trust`. Смена модели — через `Ctrl+M` (ModelSelector).

## Кейс 09: Status bar — отображение параметров

1. `.soba -i --model deepseek-chat --lang en --theme graphite`

**Критерий PASS:** Все три значения совпадают. Формат — `Provider / Model`.

## Кейс 20: Многократный /clear и отправка

**Шаги:**
1. Отправить "Тест 1"
2. `/clear`
3. Отправить "Тест 2"
4. `/clear`
5. Отправить "Тест 3"

**Ожидаемый результат:** Все 3 сообщения обработаны, хотя transcript очищался.

**Критерий PASS:** Сессия содержит все 3 turns.
