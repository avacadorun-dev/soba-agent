# Регресс-кейсы: Дерево сессии и ветвление

## Цель
Проверить древовидную структуру сессии (id/parentId), создание веток при rewind.

## Окружение
- `.soba -i` или one-shot с checkpoint'ами

---

## Кейс 01: Базовая структура (linear)

**Шаги:**
1. `.soba "Первый запрос"`
2. `.soba -c "Второй запрос"`
3. `.soba -c "Третий запрос"`
4. Прочитать JSONL

**Ожидаемый результат:** Каждый item имеет `parentId`, указывающий на предыдущий. Линейная цепочка.

**Критерий PASS:** parentId chain: first → second → third.

---

## Кейс 02: Ветвление через rewind

**Шаги:**
1. `.soba -i`
2. Отправить "Первый"
3. Поставить checkpoint (через `/rewind` или tool call)
4. Отправить "Второй (ветка A)"
5. `/rewind <checkpoint-id>`
6. Отправить "Второй (ветка B)"
7. Проверить JSONL

**Ожидаемый результат:** Две ветки:
- Первый → Второй (ветка A)
- Первый (через checkpoint) → Второй (ветка B)

**Критерий PASS:** parentId у "Второй (ветка B)" указывает на checkpoint, а не на "Второй (ветка A)".

---

## Кейс 03: Продолжение после rewind с `-c`

**Шаги:**
1. Выполнить Кейс 02
2. `.soba -c "Третий"`

**Ожидаемый результат:** Продолжается последняя активная ветка (ветка B).

**Критерий PASS:** parentId → последний item ветки B.

---

## Кейс 04: Переключение между ветками

**Шаги:**
1. Выполнить Кейс 02
2. Найти checkpoint ID
3. `.soba -c "Продолжи ветку A"` (предполагая, что активна ветка B)
4. `/rewind <checkpoint-id>`
5. `.soba -c "Продолжи ветку A снова"`

**Ожидаемый результат:** Новая ветка A2 от checkpoint.

**Критерий PASS:** В JSONL 3+ ветки.

---

## Кейс 05: Множественные ветки (5+)

**Шаги:**
1. Создать checkpoint
2. Создать 5 веток через rewind
3. Проверить JSONL

**Ожидаемый результат:** Дерево с 5 ветками.

**Критерий PASS:** Все ветки сохранены, ни одна не потеряна.

---

## Кейс 06: `/rewind` показывает все checkpoint'ы

**Шаги:**
1. Создать 3 checkpoint'а
2. `/rewind`

**Ожидаемый результат:** Список из 3 checkpoint'ов с ID и описанием.

**Критерий PASS:** Все 3 видны.

---

## Кейс 07: Проверка parentId в JSONL

**Шаги:**
1. Выполнить 3-4 запроса с ветвлением
2. Прочитать JSONL, проверить parentId каждого item

**Ожидаемый результат:** Каждый item имеет `parentId`, который существует в том же файле.

**Критерий PASS:** Все parentId ссылаются на существующие item'ы.

---

## Кейс 08: No orphan items

**Шаги:**
1. Выполнить 10+ запросов с ветвлением
2. Проверить JSONL на orphan items (нет parentId, кроме root)

**Ожидаемый результат:** Только root item может не иметь parentId.

**Критерий PASS:** Все остальные имеют parentId.
