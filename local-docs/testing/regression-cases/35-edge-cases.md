# Регресс-кейсы: Edge Cases — граничные случаи

## Цель
Проверить поведение при нестандартных и граничных условиях: отсутствие сети, повреждённые файлы, race conditions.

## Окружение
- `.soba` собран
- Возможность симулировать ошибки

---

## Кейс 01: Нет сети (airplane mode)

**Шаги:**
1. Отключить Wi-Fi/сеть
2. `.soba "Тест"`

**Ожидаемый результат:** Сообщение: "Connection failed" или "Network error".

**Критерий PASS:** Graceful error, exit != 0.

---

## Кейс 02: Нет сети, есть кеш (если реализован)

**Шаги:**
1. Отключить сеть
2. `.soba -c`

**Ожидаемый результат:** Если есть кеш сессии — может работать в офлайн.

**Критерий PASS:** Не падает.

---

## Кейс 03: API возвращает HTML вместо JSON

**Шаги:**
1. `.soba --base-url https://example.com --api-key test --model test "hi"`

**Ожидаемый результат:** Ошибка парсинга ответа.

**Критерий PASS:** Graceful error.

---

## Кейс 04: API таймаут (>60s)

**Шаги:**
1. Использовать очень медленный endpoint
2. `.soba "Тест"`

**Ожидаемый результат:** Таймаут соединения.

**Критерий PASS:** Сообщение о таймауте.

---

## Кейс 05: JSONL повреждён (broken JSON)

**Шаги:**
1. Повредить строку в JSONL сессии (добавить мусор)
2. `.soba -s <id> "Тест"`

**Ожидаемый результат:** Ошибка чтения сессии.

**Критерий PASS:** Graceful error.

---

## Кейс 06: JSONL повреждён (битые данные)

**Шаги:**
1. Записать бинарные данные в JSONL
2. `.soba -s <id>`

**Ожидаемый результат:** Ошибка чтения.

**Критерий PASS:** Graceful.

---

## Кейс 07: Конфиг с нечитаемыми правами

**Шаги:**
1. `chmod 000 ~/.soba/config.json`
2. `.soba "Тест"`

**Ожидаемый результат:** Ошибка чтения config.

**Критерий PASS:** Сообщение об ошибке.

---

**Шаги:**

2. `.soba "Создай файл"`

**Ожидаемый результат:** Ошибка записи.

**Критерий PASS:** Сообщение.

---

## Кейс 09: Нет прав на запись в ~/.soba/

**Шаги:**
1. `chmod 000 ~/.soba/`
2. `.soba "Тест"`

**Ожидаемый результат:** Ошибка создания сессии.

**Критерий PASS:** Сообщение.

---

## Кейс 10: Ctrl+C в момент записи JSONL

**Шаги:**
1. Запустить one-shot
2. Во время ответа нажать Ctrl+C

**Ожидаемый результат:** JSONL может быть неполным (корректный хвост). При загрузке — восстановление.

**Критерий PASS:** Не падает.

---

## Кейс 11: Race condition: два SOBA одновременно

**Шаги:**
1. Запустить SOBA в одном окне
2. Запустить SOBA в другом окне (та же директория)

**Ожидаемый результат:** Каждый работает со своей сессией.

**Критерий PASS:** Нет конфликта.

---

## Кейс 12: Огромный промпт (>32K символов)

**Шаги:**
1. `.soba "<prompt with 40000 chars>"`

**Ожидаемый результат:** Промпт принят (может быть обрезан или обработан).

**Критерий PASS:** Не падает.

---

## Кейс 13: Пустой ответ от API (no content)

**Шаги:**
1. Мок: API возвращает 200 с пустым телом

**Ожидаемый результат:** Сообщение об ошибке.

**Критерий PASS:** Graceful.

---

## Кейс 14: Специальные символы в session ID

**Шаги:**
1. Создать сессию с ID, содержащим `/../` (path traversal)
2. `.soba -s <id>`

**Ожидаемый результат:** ID валидируется, path traversal blocked.

**Критерий PASS:** Безопасность.

---

## Кейс 15: Несколько shell-команд подряд

**Шаги:**
1. `!echo 1 && echo 2 && echo 3`

**Ожидаемый результат:** Вывод: 1, 2, 3.

**Критерий PASS:** Все выполнены.

---

## Кейс 16: `!echo $((1+1))` — complex shell

**Шаги:**
1. `!echo $((1+1))`

**Ожидаемый результат:** Вывод: 2.

**Критерий PASS:** Команда выполнена.
