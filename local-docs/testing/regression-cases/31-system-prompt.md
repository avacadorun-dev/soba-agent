# Регресс-кейсы: Системный промпт

## Цель
Проверить формирование системного промпта, инъекцию context capsule, skills, настроек.

## Окружение
- `.soba` с `--debug`

---

## Кейс 01: System prompt содержит базовую структуру

**Шаги:**
1. `.soba --debug "Тест"`
2. Проверить debug-логи

**Ожидаемый результат:** System prompt содержит:
- Роль агента
- Инструкции по инструментам
- Формат ответов

**Критерий PASS:** Базовая структура.

---

## Кейс 02: System prompt адаптирован под язык

**Шаги:**
1. `.soba --debug --lang ru "Тест"`
2. Проверить debug-логи

**Ожидаемый результат:** System prompt на русском.

**Критерий PASS:** Язык.

---

## Кейс 03: System prompt содержит compact capsule (после компакции)

**Шаги:**
1. `.soba -i --debug`
2. Выполнить несколько turns
3. `/compact Сохрани цель тест`
4. Проверить лог следующего запроса

**Ожидаемый результат:** System prompt содержит capsule summary.

**Критерий PASS:** Capsule в промпте.

---

## Кейс 04: System prompt содержит активный skill

**Шаги:**
1. `.soba --debug -i`
2. `/skill:git-summary ...`

**Ожидаемый результат:** System prompt содержит полный SKILL.md.

**Критерий PASS:** Skill инъецирован.

---

## Кейс 05: System prompt содержит модель/настройки

**Шаги:**
1. `.soba --debug --model deepseek-chat --max-output-tokens 100 "Тест"`

**Ожидаемый результат:** System prompt или параметры содержат max_tokens info.

**Критерий PASS:** Настройки в промпте.

---

## Кейс 06: System prompt не дублируется

**Шаги:**
1. `.soba -i --debug`
2. `/compact`
3. Проверить, что prompt не содержит дубликатов capsule

**Критерий PASS:** Нет дублирования.

---

## Кейс 07: Size-limited system prompt (не превышает лимит)

**Шаги:**
1. Проверить размер system prompt

**Ожидаемый результат:** Не превышает разумный лимит (например, 8K токенов).

**Критерий PASS:** Размер < 8K.
