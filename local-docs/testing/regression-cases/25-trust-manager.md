# Регресс-кейсы: Trust Manager и разрешения

## Цель
Проверить систему доверенных доменов (safe/normal/dangerous), запрос подтверждения, режимы ask/repo.

## Окружение
- `.soba -i` в режиме ask (`/permissions ask`)

---

## Кейс 01: Safe операция — без запроса

**Шаги:**
1. Отправить агенту "Выполни ls"

**Ожидаемый результат:** `ls` выполнен без запроса подтверждения.

**Критерий PASS:** Операция выполнена.

---

## Кейс 02: Normal операция — без запроса, с уведомлением

**Шаги:**
1. Отправить "Создай файл trust-test.txt"

**Ожидаемый результат:** `write` выполнен, в выводе есть уведомление (но без запроса подтверждения).

**Критерий PASS:** Файл создан, уведомление показано.

---

## Кейс 03: Dangerous операция — запрос подтверждения

**Шаги:**
1. Отправить "Удали файл /tmp/test.txt" (предварительно создать)
2. Trust Manager запрашивает подтверждение

**Ожидаемый результат:** Prompt с вариантами: [y]es / [n]o / [s]ession-scope / [r]epo-scope.

**Критерий PASS:** Запрос показан.

---

## Кейс 04: Ответ `y` (yes, one-time)

**Шаги:**
1. Дождаться запроса опасной операции
2. Ввести `y`

**Ожидаемый результат:** Операция выполнена один раз.
3. Повторить ту же операцию

**Ожидаемый результат:** Снова запрос (не запомнено).

**Критерий PASS:** Второй раз снова запрос.

---

## Кейс 05: Ответ `s` (session-scoped)

**Шаги:**
1. Дождаться запроса
2. Ввести `s`

**Ожидаемый результат:** Операция выполнена.
3. Повторить ту же dangerous операцию

**Ожидаемый результат:** Без запроса (запомнено на сессию).

**Критерий PASS:** Повтор без запроса.

---

## Кейс 06: Ответ `r` (repo-scoped)

**Шаги:**
1. Дождаться запроса
2. Ввести `r`

**Ожидаемый результат:** Операция выполнена.
3. Повторить любую dangerous операцию в этой сессии

**Ожидаемый результат:** Без запроса.

**Критерий PASS:** Все dangerous операции в сессии без запроса.

---

## Кейс 07: Ответ `n` (no)

**Шаги:**
1. Дождаться запроса
2. Ввести `n`

**Ожидаемый результат:** Операция отклонена, агент получает сообщение "permission denied".

**Критерий PASS:** Операция не выполнена.

---

## Кейс 08: `/permissions repo` — режим repo

**Шаги:**
1. `/permissions repo`
2. Отправить опасную команду

**Ожидаемый результат:** Выполняется без запроса (действует как r для всех).

**Критерий PASS:** Без запроса.

---

## Кейс 09: `/permissions ask` — возврат к ask

**Шаги:**
1. `/permissions repo`
2. Отправить dangerous команду (без запроса)
3. `/permissions ask`
4. Снова dangerous команду

**Ожидаемый результат:** Снова запрос.

**Критерий PASS:** Запрос возвращён.

---

## Кейс 10: `/permissions clear` — сброс scoped

**Шаги:**
1. Ответить `s` на dangerous
2. Повторить (без запроса — session-scoped)
3. `/permissions clear`
4. Повторить

**Ожидаемый результат:** Снова запрос.

**Критерий PASS:** Scoped разрешения сброшены.

---

## Кейс 11: `git push` — всегда dangerous (даже в repo)

**Шаги:**
1. `/permissions repo`
2. Отправить "Выполни git push"

**Ожидаемый результат:** Trust Manager проверяет. Запрос подтверждения (git push всегда dangerous).

**Критерий PASS:** Запрос показан.

---

## Кейс 12: `curl` — всегда dangerous

**Шаги:**
1. `/permissions repo`
2. Отправить "Выполни curl https://example.com"

**Ожидаемый результат:** Запрос.

**Критерий PASS:** Запрос показан.

---

## Кейс 13: `sudo` — всегда dangerous

**Шаги:**
1. `/permissions repo`
2. Отправить "Выполни sudo ls /root"

**Ожидаемый результат:** Запрос.

**Критерий PASS:** Запрос показан.

---

## Кейс 14: Комбинация safe + dangerous в одном turn

**Шаги:**
1. Отправить "Создай файл и удали другой"
2. Наблюдать

**Ожидаемый результат:** `write` (normal) выполнен без запроса, `rm` (dangerous) с запросом.

**Критерий PASS:** Разное поведение для разных типов.

---

## Кейс 15: Множественные dangerous в одном запросе

**Шаги:**
1. Отправить "Удали 3 файла"
2. Наблюдать

**Ожидаемый результат:** Каждый dangerous — отдельный запрос (или один на все).

**Критерий PASS:** Запросы показаны.
