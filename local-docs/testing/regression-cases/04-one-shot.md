# Регресс-кейсы: One-shot режим

## Цель
Проверить выполнение разовых запросов (без интерактивного TUI) — базовый вызов, tool calls, завершение.

## Окружение
- Собранный бинарник
- Валидный config.json
- Чистая рабочая директория (временная)

---

## Кейс 01: Простой текстовый запрос

**Шаги:**
1. `.soba "Ответь одним словом: приветствие по-русски"`

**Ожидаемый результат:** Ответ одним словом (например, "Здравствуйте").

**Критерий PASS:** Ответ не пустой, содержит одно слово.

---

## Кейс 02: Запрос с чтением файла

**Шаги:**
1. `.soba "Прочитай package.json и скажи, какая там версия"`

**Ожидаемый результат:** Агент вызывает `read`, читает package.json, называет версию.

**Критерий PASS:** Ответ содержит номер версии из package.json.

---

## Кейс 03: Запрос с созданием файла

**Шаги:**
1. `.soba "Создай файл hello.txt с содержимым Hello World"`

**Ожидаемый результат:** Файл `hello.txt` создан с содержимым "Hello World".

**Критерий PASS:** `cat hello.txt` == "Hello World".

---

## Кейс 04: Запрос с редактированием файла

**Шаги:**
1. Создать `test.txt` с содержимым "Старый текст"
2. `.soba "Измени test.txt: замени 'Старый' на 'Новый'"`

**Ожидаемый результат:** `test.txt` содержит "Новый текст".

**Критерий PASS:** `cat test.txt` == "Новый текст".

---

## Кейс 05: Запрос с выполнением bash

**Шаги:**
1. `.soba "Выполни ls -la и кратко опиши, что видишь"`

**Ожидаемый результат:** Агент вызывает `bash ls -la`, описывает содержимое директории.

**Критерий PASS:** Ответ содержит список файлов этой директории.

---

## Кейс 06: Множественные tool call'ы

**Шаги:**
1. `.soba "Создай файл a.txt с текстом A, затем файл b.txt с текстом B, затем прочитай оба"`

**Ожидаемый результат:** Три tool call'а: write + write + read. Ответ содержит содержимое обоих.

**Критерий PASS:** Файлы существуют, ответ корректен.

---

## Кейс 07: One-shot с --no-session

**Шаги:**
1. `.soba --no-session "Тест"`
2. Проверить `~/.soba/sessions/`

**Ожидаемый результат:** Файл сессии не создан.

**Критерий PASS:** Нет новых файлов в sessions/.

---

## Кейс 08: One-shot с --max-agent-iterations 1

**Шаги:**
1. `.soba --max-agent-iterations 1 "Создай файл x.txt с текстом X, затем прочитай его"`

**Ожидаемый результат:** Агент выполняет только 1 tool call (или завершает без второго).

**Критерий PASS:** Может быть не более 1 tool call.

---

## Кейс 09: One-shot с --budget 100

**Шаги:**
1. `.soba --budget 100 "Напиши очень длинный ответ"`

**Ожидаемый результат:** Агент останавливается по превышению бюджета.

**Критерий PASS:** Сообщение о превышении бюджета.

---

## Кейс 10: One-shot с запросом checkpoint

**Шаги:**
1. `.soba "Поставь checkpoint с описанием 'тестовый checkpoint'"`

**Ожидаемый результат:** Агент вызывает `checkpoint`, checkpoint создан в сессии.

**Критерий PASS:** В JSONL сессии есть item с type=checkpoint.

---

## Кейс 11: One-shot с активацией skill

**Шаги:**

**Ожидаемый результат:** Skill активирован, выполнен.

**Критерий PASS:** Ответ содержит git-анализ.

---

## Кейс 12: One-shot с ошибкой в tool call

**Шаги:**
1. `.soba "Прочитай файл nonexistent-file-12345.txt"`

**Ожидаемый результат:** Агент сообщает, что файл не найден.

**Критерий PASS:** Ответ содержит сообщение об ошибке чтения.

---

## Кейс 13: One-shot на русском языке

**Шаги:**
1. `.soba --lang ru "Напиши одно слово: приветствие"`

**Ожидаемый результат:** Ответ на русском.

**Критерий PASS:** Слово на русском.

---

## Кейс 14: One-shot с длинным промптом (>1000 символов)

**Шаги:**
1. Создать промпт из 1500 символов (повторяющийся текст)
2. `.soba "<длинный промпт>"`

**Ожидаемый результат:** Промпт обработан, ответ получен.

**Критерий PASS:** Ответ не пустой.

---

## Кейс 15: One-shot — exit code 0

**Шаги:**
1. `.soba "echo done"` (если агент выполняет bash, или просто "ok")

**Ожидаемый результат:** exit code 0.

**Критерий PASS:** `echo $?` === 0.

---

## Кейс 16: One-shot без промпта (пустая строка)

**Шаги:**
1. `.soba ""`

**Ожидаемый результат:** Сообщение об ошибке или пустой ответ.

**Критерий PASS:** Не падает.

---

## Кейс 17: One-shot с env var `SOBA_MODEL`

**Шаги:**
1. `SOBA_MODEL=deepseek-chat .soba "Ответь одним словом: привет"`

**Ожидаемый результат:** Ответ от DeepSeek / deepseek-chat (активный провайдер из конфига).

**Критерий PASS:** Ответ не пустой.

---

## Кейс 18: One-shot после `soba provider use`

**Шаги:**
1. `soba provider use openrouter`
2. `.soba --model openai/gpt-4o-mini "Ответь одним словом: привет"`

**Ожидаемый результат:** Ответ от OpenRouter с указанной моделью.

**Критерий PASS:** Ответ не пустой.
