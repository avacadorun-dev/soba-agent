# Регресс-кейсы: Agent Loop

## Цель
Проверить работу цикла агента — multi-step reasoning, ограничение итераций, stall recovery, таймауты.

## Окружение
- `.soba` собран
- API endpoint доступен

---

## Кейс 01: Базовый цикл (один tool call)

**Шаги:**
1. `.soba "Создай файл loop-test.txt с текстом pass"`

**Ожидаемый результат:** Агент выполняет 1 вызов `write` → получает результат → завершает.

**Критерий PASS:** Файл создан, агентн завершился с финальным ответом.

---

## Кейс 02: Множественные tool call'ы в одном turn

**Шаги:**
1. `.soba "Создай файл a.txt, прочитай его, создай b.txt"`

**Ожидаемый результат:** Агент в одной итерации может запланировать/выполнить несколько tool call'ов (зависит от модели).

**Критерий PASS:** Все три операции выполнены.

---

## Кейс 03: `--max-agent-iterations 2`

**Шаги:**
1. `.soba --max-agent-iterations 2 "Создай файл 1.txt, прочитай его, создай 2.txt, прочитай его, создай 3.txt"`

**Ожидаемый результат:** Агент выполняет не более 2 итераций (2 tool call + response).

**Критерий PASS:** Не более 2 tool call'ов, сообщение о лимите итераций.

---

## Кейс 04: `--max-agent-iterations 0` (без лимита)

**Шаги:**
1. `.soba --max-agent-iterations 0 "Создай 3 файла: 1.txt, 2.txt, 3.txt"`

**Ожидаемый результат:** Все 3 файла созданы, нет сообщения о лимите.

**Критерий PASS:** 3 файла созданы.

---

## Кейс 05: `--max-stalled-iterations 2`

**Шаги:**
1. `.soba --max-stalled-iterations 2 "Повторяй одно и то же действие снова и снова"`

**Ожидаемый результат:** После 2 stalled-итераций срабатывает stall recovery. (Трудно воспроизвести — нужен мок.)

**Критерий PASS:** Сообщение о stall recovery.

---

## Кейс 06: `--max-run-minutes 0.1` (6 секунд)

**Шаги:**
1. `.soba --max-run-minutes 0.1 "Выполни: echo start && sleep 10 && echo end"`

**Ожидаемый результат:** Процесс прерывается через ~6 секунд.

**Критерий PASS:** Сообщение о таймауте.

---

## Кейс 07: `--max-run-minutes 0` (без лимита)

**Шаги:**
1. `.soba --max-run-minutes 0 "Создай файл time-test.txt"`

**Ожидаемый результат:** Без ограничения по времени, файл создан.

**Критерий PASS:** Файл создан.

---

## Кейс 08: Completion Gate — полный ответ

**Шаги:**
1. `.soba "Ответь 'готово'"`

**Ожидаемый результат:** Ответ "готово" — gate помечает как complete.

**Критерий PASS:** Цикл завершён.

---

## Кейс 09: Completion Gate — incomplete ответ

**Шаги:**
1. Если модель отдаёт `stop_reason: "max_tokens"`, gate должен продолжить цикл

**Ожидаемый результат:** Gate детектирует incomplete и запрашивает продолжение.

**Критерий PASS:** Цикл продолжается.

---

## Кейс 10: Loop Guard — зацикливание

**Шаги:**
1. `.soba --max-stalled-iterations 3 "Просто продолжай повторять: я зациклился"`

**Ожидаемый результат:** Loop Guard детектирует stall после N итераций.

**Критерий PASS:** Guard сработал, сообщение.

---

## Кейс 11: Нет tool call'ов (чисто текстовый ответ)

**Шаги:**
1. `.soba "Ответь одним словом: ok"`

**Ожидаемый результат:** Агент отвечает "ok" без tool call'ов, цикл завершён за 1 итерацию.

**Критерий PASS:** Только 1 итерация (message → response).

---

## Кейс 12: Tool call + ошибка + повтор

**Шаги:**
1. `.soba "Прочитай файл nonexistent.txt, если ошибка — создай его"`

**Ожидаемый результат:** Агент читает → ошибка → создаёт файл.

**Критерий PASS:** 2 tool call'а (read + write).

---

## Кейс 13: Длинная цепочка (5+ tool call'ов)

**Шаги:**
1. `.soba "Создай 5 файлов: 1.txt, 2.txt, 3.txt, 4.txt, 5.txt"`

**Ожидаемый результат:** Все 5 файлов созданы.

**Критерий PASS:** 5 tool call'ов write.

---

## Кейс 14: Агент сам решает, когда завершить

**Шаги:**
1. `.soba "Проверь git status, затем создай файл status.txt с результатом"`

**Ожидаемый результат:** Агент выполняет bash (git status) → write (status.txt) → завершает.

**Критерий PASS:** 2 tool call'а, финальный ответ.
