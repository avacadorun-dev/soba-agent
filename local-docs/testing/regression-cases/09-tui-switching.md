# Регресс-кейсы: TUI — смена модели, провайдера, языка, темы

## Цель
Проверить runtime-переключение модели (через ModelSelector с discovery, без хардкода),
языка интерфейса и темы оформления.

## Окружение
- TUI запущен (`soba -i`)

---

## Кейс 01: `Ctrl+M` — открывает ModelSelector с провайдерами и моделями

**Шаги:**
1. Нажать `Ctrl+M`
2. ModelSelector показывает список провайдеров (DeepSeek, Kimi, Alibaba, OpenRouter + custom)

**Ожидаемый результат:** Оверлей открыт, видны все провайдеры. У built-in провайдеров
модели появляются после discovery или показывается placeholder.

**Критерий PASS:** ModelSelector открыт, провайдеры перечислены.

---

## Кейс 02: ModelSelector — поиск и выбор модели

**Шаги:**
1. Нажать `Ctrl+M`
2. Ввести поисковый запрос (например, "coder")
3. Стрелками выбрать модель
4. Нажать Enter

**Ожидаемый результат:** Модель переключена, status bar обновлён, оверлей закрыт.

**Критерий PASS:** Модель переключена, следующий turn использует новую модель.

---

## Кейс 03: ModelSelector — нет хардкод-моделей, discovery с `/v1/models`

**Шаги:**
1. Нажать `Ctrl+M`
2. Выбрать built-in провайдера (например, Kimi)
3. Если discovery ещё не сделан — SOBA делает `GET /v1/models`
4. Показать список из живого API

**Ожидаемый результат:** Список моделей соответствует `/v1/models`, а не хардкод-списку.

**Критерий PASS:** Модели из API, нет единственного хардкоднутого варианта.

---

## Кейс 04: ModelSelector — несуществующая модель (typed manually)

**Шаги:**
1. Нажать `Ctrl+M`
2. Ввести поисковый запрос "nonexistent-model-xyz"
3. Нажать Enter (или выбрать, если попадёт в synthetic entry)

**Ожидаемый результат:** API возвращает ошибку (модель не найдена), но TUI не падает.
Модель меняется на введённую (synthetic entry), ошибка при следующем turn.

**Критерий PASS:** Ошибка отображена, TUI работает.

---

## Кейс 05: `Ctrl+M` во время активного turn

**Шаги:**
1. Отправить агента "Напиши длинный текст"
2. Во время ответа нажать `Ctrl+M`

**Ожидаемый результат:** ModelSelector открывается, текущий turn завершается со старой моделью.

**Критерий PASS:** Следующий turn использует новую модель.
