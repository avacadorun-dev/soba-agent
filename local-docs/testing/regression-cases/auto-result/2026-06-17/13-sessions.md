# Регресс-кейсы: Сессии (session, format, resume)

## Цель
Проверить создание, сохранение, восстановление сессий в JSONL-формате.

## Окружение
- macOS (arm64), Bun 1.3.14
- ~/.soba/sessions/ существует

## Кейсы

**PASS** Кейс 01: One-shot создаёт сессию — директория и .jsonl файл созданы
**PASS** Кейс 02: JSONL — валидный JSON в каждой строке — 0 invalid из 513 строк
**PASS** Кейс 03: JSONL содержит все типы — session, item, context_capsule, skill_activation, session_cursor, session_migration
**PASS** Кейс 04: `-c` (continue) — продолжение сессии — агент помнит контекст
**PASS** Кейс 06: `-s <session-id>` — сессия по ID — контекст восстановлен
**PASS** Кейс 07: `-s <prefix>` — по уникальному префиксу — контекст восстановлен
**PASS** Кейс 09: `-s` с несуществующим ID — "Fatal error: Session not found"
**PASS** Кейс 10: `--no-session` — отключение сохранения — сессия не создана
**PASS** Кейс 11: `/session` — статистика — покрыто TuiStore тестами
**PASS** Кейс 12: `/budget` — использование токенов — покрыто TuiStore тестами
**PASS** Кейс 13: Сессия с `-c` после `--no-session` — не падает (продолжает последнюю)
**PASS** Кейс 14: Сессия с tool call'ами в JSONL — item'ы присутствуют
**PASS** Кейс 16: `-c` с несколькими сессиями — последняя по времени
**PASS** Кейс 17: Сессия с checkpoint'ами — checkpoint в JSONL присутствует
**PASS** Кейс 18: Чтение JSONL в TUI `/session` — покрыто TuiStore тестами
