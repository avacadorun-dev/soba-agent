# Регресс-кейсы: CLI-флаги и аргументы

## Цель
Проверить все флаги командной строки и их комбинации.

## Окружение
- Собранный бинарник (`bun run build`)
- Существующий `~/.soba/config.json` с валидным API key
- Реальный OpenAI-compatible endpoint (или мок с подменой ответов)

---

## Кейс 01: `--version` — версия

**Шаги:**
1. `./dist/bin/soba-darwin-arm64 --version`

**Ожидаемый результат:** `soba v0.3.2`

**Критерий PASS:** Точное совпадение строки.

---

## Кейс 02: `--help` — справка

**Шаги:**
1. `./dist/bin/soba-darwin-arm64 --help`

**Ожидаемый результат:** Вывод списка всех флагов с описанием.

**Критерий PASS:** Содержит Usage, Options, Commands, все флаги.

---

## Кейс 03: `--model` — смена модели (provider-aware)

**Шаги:**
1. `.soba --model deepseek-chat "Ответь 'ok'"`

**Ожидаемый результат:** Модель `deepseek-chat` из активного провайдера (по умолчанию DeepSeek). Если модель найдена в одном из built-in провайдеров, она выбирается — иначе используется как есть (synthetic entry).

**Критерий PASS:** Ответ не пустой, сообщение обработано.

---

## Кейс 04: `--model` без аргумента

**Шаги:**
1. `.soba --model "Привет"`

**Ожидаемый результат:** Используется модель из config (default model активного провайдера), "Привет" — промпт.

**Критерий PASS:** Не интерпретируется как пустой model + "Привет" как значение.

**Критерий PASS:** Exit code 0, провайдер добавлен и сохранён в конфиг.
