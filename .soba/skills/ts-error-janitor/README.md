# TypeScript Error Janitor

Скилл для автоматического массового исправления накопившихся TypeScript-ошибок в проекте SOBA Agent.

## Проблема

Когда в проекте долго не включён `tsc --noEmit`, ошибки копятся снежным комом. Одна пропущенная `.d.ts` декларация порождает 50+ каскадных ошибок внизу. Ручной фикс — дни работы. Этот скилл делает это итеративно, с верификацией и прогрессом.

## Как это работает

**Не «всё разом».** Модель получает 200 ошибок — теряет контекст, ломает больше, чем чинит.

**Водопад с обратной связью:**
1. `bunx tsc --noEmit` → собираем ошибки
2. Группируем по приоритету (P0–P5)
3. Берём порцию 1–5 файлов
4. Фиксим → проверяем `tsc` → фиксим следующую порцию

## Структура

```
.soba/skills/ts-error-janitor/
├── SKILL.md          # Полный протокол работы агента
└── progress.json     # Текущий прогресс и история фаз

docs/testing/ts-errors/
├── .gitignore        # Игнорируем архивные логи tsc
└── *.txt             # Логи ошибок (генерируются автоматически)
```

## Приоритеты ошибок

| Приоритет | Тип | Почему первым |
|-----------|-----|---------------|
| P0 | `TS2304` Cannot find name | Одна дыра в типах → каскад из 50 ошибок |
| P1 | `TS2307` Cannot find module | Ломает целые деревья импортов |
| P2 | `TS2322` Type not assignable | Часто следствие P0/P1, иногда реальная логика |
| P3 | `TS2345` Argument type mismatch | Локальные ошибки в вызовах функций |
| P4 | `TS7006` Implicit any | Strict-mode, не ломает компиляцию |
| P5 | `TS6133` Unused variable | Чистка, можно отложить |

## Запуск

```
Почини TS ошибки
```

или

```
Run TS janitor
```

Агент сам:
- Запустит `tsc`, проанализирует вывод
- Выберет первую порцию по приоритету
- Исправит файлы
- Проверит `tsc` и `bun test`
- Сохранит прогресс в `progress.json`

## Безопасность

- **Не удаляет файлы** — только правит типы
- **Не меняет логику** — если нужен рефакторинг, уходит в `skipped` с тикетом
- **Не использует `@ts-ignore` массово** — максимум 1–2 на проект
- **Всегда `bun test` после фазы** — типы ≠ работающий код

## Когда остановиться

- **0 ошибок** — идеал. Добавить `tsc --noEmit` в pre-commit.
- **Остались только P4–P5** — можно остановиться, это strict-проверки.
- **3 цикла без прогресса** — требуется рефакторинг, не типофикс. Остановить, создать тикет.

## После завершения

Добавь в `package.json`:
```json
"scripts": {
  "typecheck": "tsc --noEmit"
}
```

И в CI / pre-commit:
```bash
bun run typecheck
```

Чтобы снежный ком не вернулся.

## Требования

- Bun runtime
- `tsc` в `devDependencies`
- Git (для отката при неудачной порции)
