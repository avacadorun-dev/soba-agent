# Regression Validator — Валидация регресс-кейсов перед прогоном

Проходит по файлам регресс-кейсов в `docs/testing/regression-cases/` и проверяет их на соответствие текущей реализации. Выявляет расхождения: несуществующие тестовые файлы, удалённые CLI-флаги, переименованные функции, неактуальные кейсы.

> **Dev-only** — этот скилл только для разработки SOBA Agent. Не попадает в bundled скилы продукта.

## Расположение

- **Скилл**: `.soba/skills/regression-validator/SKILL.md`
- **Регресс-кейсы (вход)**: `docs/testing/regression-cases/`
- **Результаты (выход)**: `docs/testing/regression-cases/auto-result/validation/YYYY-MM-DD/`
- **Сводный отчёт**: `VALIDATION-SUMMARY.md`

## Быстрый старт

Скажи агенту:

```
Свалидируй регресс
```

или

```
Validate regression cases
```

или для конкретного файла:

```
Проверь кейсы 13
```

Агент автоматически:
1. Проходит по всем регресс-файлам (или указанному)
2. Извлекает все ссылки: тестовые файлы, CLI-флаги, инструменты, slash-команды, исходники
3. Проверяет существование каждого элемента в кодовой базе
4. Классифицирует расхождения (VALID / MISMATCH / STALE / BROKEN)
5. Ищет пробелы в покрытии (тесты/модули без кейсов)
6. Сохраняет результаты и показывает сводку
7. Оценивает готовность к запуску regression-runner

## Что проверяется

| Категория | Как проверяется | Пример |
|-----------|----------------|--------|
| **Тестовые файлы** | `ls tests/<path>` | `tests/session-manager.test.ts` |
| **CLI-флаги** | `--help` + `grep src/cli/` | `--model`, `--continue`, `--session` |
| **Инструменты** | `grep src/core/tools/` | `ls`, `read`, `write`, `edit`, `bash` |
| **Slash-команды** | `grep src/tui/` | `/help`, `/clear`, `/budget`, `/session` |
| **Исходники** | `ls src/<path>` | `src/core/session/session-manager.ts` |
| **Версия** | `--version` / `package.json` | `v0.3.2` |
| **Конфиг-ключи** | `grep src/config/` | `apiKey`, `baseUrl`, `model` |

## Что НЕ проверяется

- Проходят ли тесты — это задача **regression-runner**
- Корректность поведения — это задача **regression-runner**
- API-ответы — это задача **regression-runner**
- Рендеринг TUI — это задача **regression-runner**

## Вердикты

| Вердикт | Описание | Действие |
|---------|----------|----------|
| 🟢 **VALID** | Все проверки пройдены | ✅ Готово к прогону |
| 🟡 **MINOR** | Незначительные расхождения (версия, путь) | ⚠️ Можно прогонять, но обновить |
| 🟠 **STALE** | Устаревшие ссылки (удалён флаг/инструмент) | 🔴 Исправить перед прогоном |
| 🔴 **BROKEN** | Большинство ссылок битые | 🔴 Переписать кейс |

## Структура результатов

```
docs/testing/regression-cases/auto-result/validation/YYYY-MM-DD/
├── 01-installation-build.md      # Результаты валидации файла 01
├── 02-cli-flags.md               # Результаты валидации файла 02
├── ...
└── VALIDATION-SUMMARY.md         # Сводный отчёт
```

## Интеграция с regression-runner

```
regression-validator → regression-runner → bug-fixer
         │                     │                │
   Валидация кейсов     Прогон кейсов    Исправление багов
   (статика)            (динамика)       (по результатам)
```

- **Validator** запускается ПЕРВЫМ — проверяет что кейсы актуальны
- **Runner** запускается ВТОРЫМ — если validator не нашёл STALE/BROKEN
- **Bug-fixer** запускается ТРЕТЬИМ — исправляет FAIL из runner

## Когда запускать

- **Перед каждым regression-runner** — обязательно
- **После рефакторинга** — проверить что кейсы не разошлись
- **После добавления/удаления фич** — найти пробелы
- **Периодически (раз в неделю)** — аудит качества кейсов
