# Bug Fixer — Скилл для автоматического исправления регресс-багов

Читает последний `SUMMARY.md` из `docs/testing/regression-cases/auto-result/`, находит FAIL-баги, верифицирует воспроизведение, фиксит код и обновляет статус.

> **Dev-only** — этот скилл только для разработки SOBA Agent. Не попадает в bundled скилы продукта.

## Расположение

- **Скилл**: `.soba/skills/bug-fixer/SKILL.md`
- **Результаты регресса**: `docs/testing/regression-cases/auto-result/YYYY-MM-DD/`
- **Код**: `src/` — куда вносятся исправления

## Быстрый старт

Скажи агенту после регресс-прогона:

```
Исправь баги из регресса
```

или

```
Fix bugs from latest regression run
```

Агент автоматически:
1. Найдёт последний SUMMARY.md в `auto-result/`
2. Прочитает FAIL-баги
3. Для каждого: воспроизведёт → проанализирует → исправит → подтвердит
4. Обновит статус в SUMMARY.md
5. Создаст коммит с исправлениями

## Когда запускать

- **После каждого регресс-прогона** — `regression run` → `fix bugs`
- **По запросу разработчика** — «пофикси баг с --context-window»
- **В CI/CD** — после автоматического регресс-прогона

## Примеры исправляемых багов

| Баг | Тип | Типичный фикс |
|-----|-----|--------------|
| `--no-color` оставляет ANSI | Вывод | Добавить `if (!config.noColor)` перед ANSI-кодами |
| `--context-window 32000` → fatal | Валидация | Исправить границы `validateCompactionConfig` или graceful error |
| `--foobar` не вызывает ошибку | Валидация аргументов | Добавить `else` ветку в парсер с `console.error` + exit |
| Любые fatal → graceful | Обработка ошибок | Заменить `process.exit` / `throw` на обработчик |
